¿Qué es un API?

La Interfaz que permite que diferentes aplicaciones se comuniquen entre sí

Definición Simple

API significa Interfaz de Programación de Aplicaciones (Application Programming Interface).

Es como un menú de restaurante que te dice qué platos puedes pedir, cómo pedirlos y qué recibirás, sin necesidad de conocer cómo se cocina cada plato.

Aplicación Cliente

Servidor / API

Un API define las reglas y protocolos que permiten que diferentes sistemas de software se comuniquen entre sí.

La Analogía del Restaurante

1
Tú (cliente) miras el menú (documentación del API)
2
Pides tu comida (envías una solicitud HTTP)
3
La cocina (servidor) prepara tu pedido (procesa la solicitud)
4
Recibes tu comida (obtienes una respuesta en JSON/XML)
¡No necesitas saber cocinar!

Así como no necesitas saber cocinar para disfrutar de un buen plato, no necesitas saber cómo funciona internamente un servicio para usar su API.

Ejemplo real: Cuando usas Google Maps en una app de delivery, esa app está usando el API de Google Maps sin tener que crear su propio sistema de mapas.

¿Cómo funciona un API?

Los APIs web normalmente funcionan sobre HTTP/HTTPS usando estos métodos:

GET /api/usuarios Obtiene lista de usuarios
POST /api/usuarios Crea un nuevo usuario
PUT /api/usuarios/123 Actualiza usuario con ID 123
DELETE /api/usuarios/123 Elimina usuario con ID 123

Flujo típico de una solicitud API:

// 1. Cliente envía solicitud HTTP
GET https://api.ejemplo.com/usuarios

// 2. Servidor procesa la solicitud
if (usuarioAutenticado) {
  return "lista de usuarios en JSON";
}

// 3. Cliente recibe respuesta
{
  "status": "success",
  "data": [ ... ]
}

Tipos de APIs y Usos Comunes

Tipos de APIs más comunes:

APIs Web (REST)

Lo más común hoy. Usan HTTP y devuelven JSON/XML.

APIs en Tiempo Real

WebSockets, GraphQL subscriptions para datos en vivo.

APIs de Biblioteca

Funciones dentro de un lenguaje (como las de JavaScript).

APIs Privadas

Para uso interno de una empresa o aplicación.

¿Dónde encuentras APIs en la vida real?

  • Pagos en línea: Stripe, PayPal
  • Mapas y geolocalización: Google Maps, Mapbox
  • Clima: OpenWeather, WeatherAPI
  • E-commerce: Shopify, WooCommerce
  • Redes sociales: Twitter, Facebook, Instagram

APIs en Microservicios

Comunicación entre Microservicios

Los APIs son el "lenguaje común" que permite a los microservicios comunicarse.

Ejemplo: El servicio de "Usuarios" expone un API que el servicio de "Pedidos" consume para verificar usuarios.

APIs Públicas vs Privadas

API Pública: Abierta a cualquier desarrollador (como Twitter API)

API Privada: Solo para uso interno de tu aplicación

API de Partners: Para socios comerciales específicos

¡Importante en Microservicios!

En una arquitectura de microservicios, cada servicio tiene su propio API bien definido. Esto permite que los servicios evolucionen independientemente mientras mantienen compatibilidad con los servicios que dependen de ellos.
